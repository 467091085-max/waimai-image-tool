# 产品化工作计划

本文档保存外卖菜品图工具从 MVP 到可售卖产品的模块计划。当前原则：

- 高质量出图默认走混元生成，旧图库只做参考或人工兜底。
- 新生成的品类背景图、免费样图、正式菜品图都必须沉淀到服务器目录或生产对象存储；AI asset manifest 必须打上品类、菜名、关键词、风格、质量、来源和存储位置等标签，后续用于复用。
- 前台产品口径不宣传“真实图库”，只表达为 AI 生成、样图预览和历史生成资产复用；历史/种子图片只作为内部参考、兜底或资产沉淀来源。
- C 端用积分激励，代理端统一按直接订单实付净额 20% 现金返佣。
- 代理和 C 端邀请默认只算一级直推；不做下级代理抽成、不做多级充值返利。任何多级分销方案都暂不启用，必须法务确认后另开方案和实现。
- 所有客户菜单、原图、生成图、导出包都必须进入权限控制和对象存储体系。

当前落地状态以 `MODULE_STATUS.md` 为准：那里区分“已落本地代码”和“仍需外部服务/生产化接入”。

## 模块总览

| 对象存储 | 菜单、原图、生成图、导出包、AI 资产统一沉淀到 COS/OSS/R2 等私有对象存储 |
| 积分/支付 | 积分账本、充值订单、扣费、退款、幂等、微信/支付宝回调 |

| 生成队列 | 批量正式出图异步化、进度、重试、取消、部分完成导出 |
| AI 图库沉淀 | 品类背景图、免费样图、正式菜品图进入 AI asset manifest 并可复用 |
| 防盗图/签名 URL | 私有桶、短期签名、下载鉴权、低清/水印预览 |
| 防刷风控 | 注册、短信、邀请、代理佣金、下载、生成频率和异常退款风控 |

## 当前本地状态同步

截至当前 worktree，本地 MVP 已补齐以下产品化子能力，但都不等同于生产上线：

- 后台 lists 明细：`/api/admin/lists/*` 已覆盖生成任务、资产访问、风控事件、佣金结算、订单等明细，后台前端已优先读取既有列表，再回退到 dashboard 汇总；`risk-events` 后端资源已可按 decision/riskLevel/eventType/userId/search/时间分页查询，前端接入另排。
- AI 资产审核动作：`/api/admin/actions/ai-assets/<asset_id>/status` 支持 approve/reject/disable 和审核备注，写入后台操作审计；仍缺完整人工审核队列、角色权限和运营工作流。
- 生成队列错误语义：正式出图前端走 `/api/generation-jobs` 异步任务；队列满返回 429 `generation_queue_full`，队列关闭或不可用返回 503 `generation_queue_unavailable`。
- 资产访问筛选：`asset-access` 明细支持 `status=denied/allowed` 等运营筛选口径，便于排查盗链、签名失败和拒绝访问。
- 风控日志筛选：`risk-events` 明细支持 `decision/riskLevel/eventType/userId/search/createdFrom/createdTo/sort/order/limit/offset`，用于后台排查注册、邀请、下载和生成相关风险判定。
- 契约测试保护：已有测试锁定前台不得回退到“真实图库/免费样图预览”等旧口径，正式出图必须走异步任务，后台必须保留 lists 明细接入。

仍未完成或未接真实外部服务：

- 手机短信、微信登录、微信/支付宝真实支付、退款补单和财务对账。
- PostgreSQL 迁移、生产迁移脚本、备份和索引复核。
- 私有 COS/OSS/R2 全链路、低清/水印预览、Redis/DB 原子一次性下载 token。
- Redis/RQ/Celery 等跨进程队列、任务恢复、失败重试和队列监控。
- 代理提现、实名/主体认证、税务信息、真实打款和完整后台权限。

## 防盗图 / 防破解图库机制

### 威胁模型

保护对象：

- AI 资产库原图、模板图、提示词素材、参考素材。
- 用户生成的菜品预览图、高清图、可商用下载图。
- 图片对象存储路径、签名 URL、订单/积分/下载权限。
- 生成接口、批量下载接口、图库搜索接口。

主要攻击者：

- 未登录用户：直接访问图片 URL、枚举对象路径。
- 普通注册用户：批量爬取预览图、绕过积分下载高清图。
- 付费用户：脚本批量生成、批量下载、共享签名链接。
- 竞争对手/爬虫：抓取图库、菜品图、样图用于二次售卖。
- 被盗账号：消耗积分、导出图库资源。
- 内部误配置：COS Bucket 公开读、对象 Key 可枚举、日志缺失。

安全目标：

- 不追求绝对 DRM，目标是显著提高盗图、批量爬取、破解图库成本。
- 原图只对有权限用户短时间开放。
- 未付费/未消耗积分前只能看到低清带水印预览。
- 所有敏感访问可追踪、可限速、可封禁。

- 未登录用户猜测 URL 批量下载生成图。
- 付费用户爬取其他用户的图片。
- 代理或客户批量抓取 AI 资产库。
- 前端泄露真实 COS key、服务器路径、资产原图 URL。
- 下载链接被转发到群里长期可用。
- 爬虫通过免费样图接口刷图或盗走样图。

### 访问控制

- 所有客户资产必须绑定 `user_id`、`store_id`、`job_id`、`order_id`。
- 图片读取走应用鉴权或短期签名 URL，不直接暴露长期公开桶地址。
- 客户只能访问自己门店/订单下的生成图和导出包。
- 代理只能看自己归属客户的业务数据，不能看客户原始图片和非授权导出包。
- 管理员访问图库和 AI 资产库必须有后台权限和审计日志。

### URL 签名

- 预览图签名有效期：5-15 分钟，MVP 默认 5 分钟。
- 受控下载签名有效期：MVP 默认 10 分钟。
- 后续可细分为高清原图 30-120 秒、导出 ZIP 30 分钟。
- 管理后台查看素材签名有效期：1-5 分钟。
- 签名内容包含：`asset_id`、`user_id`、`order_id`、`variant`、`purpose`、`expires_at`、`nonce`。
- 服务端校验 HMAC，不接受前端传来的任意 path。
- URL 过期后必须重新向服务端申请。
- 原图下载链接尽量一次性使用；本地 MVP 已有 `nonce` 消费键和 replay 判定，生产需存 Redis/DB 并在下载成功后原子标记已使用。
- 不强绑定 IP，IP/UA 作为风控信号，避免误伤移动网络和企业网络。
- 原图响应头：`Cache-Control: private, no-store`。

### 样图保护

- 免费样图只提供低清预览，最长边 512-768px。
- WebP/JPEG 中等质量，例如 60-75。
- 免费样图叠加平台水印或不可见追踪标识。
- 水印可包含平台名、用户 ID 短码、时间短码，便于追踪泄露来源。
- 未正式扣费前不提供原图下载。
- 未登录或无菜单上下文时不允许批量生成免费样图。

### 原图隔离

- `cleanpic`、客户菜单、客户上传图、AI 资产原图不得出现在前端 JSON。
- 前端只拿 `assetId` 和短期预览 URL。
- `public_plan_payload` 必须继续移除 `path`、私有 `cosKey`、内部 `remoteUrl`。
- COS bucket 默认私有读。

### 防刷和限流

- 免费样图：同用户每菜单每风格每天限制次数。
- 正式生成：按账号积分和订单状态限制。
- 下载：同订单每小时限制下载次数。
- IP、设备指纹、手机号、支付账号联合风控。
- 异常行为进入人工审核：大量重复上传、重复生成、短时批量下载。

### 审计日志

每次资产访问记录：

- `user_id`
- `agent_id`
- `asset_id`
- `job_id`
- `order_id`
- `action`: preview/download/export/admin_view
- `ip`
- `user_agent`
- `created_at`
- `allowed`
- `deny_reason`

当前本地 MVP 已接入导出下载和对象读取审计：允许、缺 token、签名失败、对象不匹配、对象不存在等都会尽量写入 `asset_access_logs`，供后台 dashboard 和 lists 明细汇总。`asset-access` 明细支持 `allowed` 和 `status=denied/allowed` 等筛选。一次性 token 已有消费键和 replay 判定规则。生产仍需日志聚合、告警、长期留存和 Redis/DB 原子消费记录。

### MVP 优先级

P0：

1. COS Bucket 改为私有读，关闭公开访问。
2. 前端不暴露原图 URL。
3. 后端鉴权后签发短期 URL。
4. 预览图和原图分路径存储。
5. 未扣积分只能看低清带水印预览。
6. 原图下载前校验用户归属和积分权益。
7. 图片 Key 改为不可枚举 UUID 路径。
8. 生成、预览、下载接口加基础限流。
9. 下载和权限拒绝写审计日志。当前本地 MVP 已覆盖导出下载和对象读取。

P1：

1. 原图下载一次性 token 持久化消费记录。
2. Redis 滑动窗口限流。
3. 高频访问触发验证码/冷却。
4. 同账号多 IP 异常下载告警。
5. 图库搜索限制分页大小和访问频率。

7. 水印加入用户短码或追踪码。

## 代理机制

### 返现还是返积分

代理端规则：现金佣金为主，统一按直接订单实付净额 20% 返佣；推广积分只可作为运营试用额度，不作为默认返佣承诺。

理由：

- 代理本质是销售渠道，需要现金收益才有长期动力。
- 纯返积分适合自用客户，不适合作为代理收入。
- 即时正反馈以待结算现金佣金为主；如运营需要演示额度，可另行发放即时到账推广积分。

建议规则：

- 代理直接客户发生有效订单后，代理后台立即显示“预计佣金”。
- 现金佣金进入待结算余额，过风控期后可提现。
- 如需给代理演示、试单或自用额度，由运营单独发放推广积分；该积分不能提现、不计入佣金、不作为多级收益依据。

### 佣金基数

代理佣金建议按客户实付金额创建“待结算佣金”，并在风控期后确认；内部成本核算可同时记录客户实际消耗积分。

原因：

- 避免客户充值后退款、薅返佣。
- 避免代理自充刷佣金。
- 平台收入和代理收益按真实消耗对齐。

公式：

```text
现金佣金基数 = 直接客户订单实付金额 - 退款金额 - 优惠券抵扣 - 平台补偿
现金佣金 = 现金佣金基数 * 20%
推广积分 = 运营试用额度，可人工发放；不能提现，不参与佣金计算
```

### 合规边界

当前版本只允许一级直推佣金：代理只能拿自己直接邀请客户的佣金，不能拿下级代理、下下级代理或间接客户的佣金。

原因：

- 中国监管口径下，“三级以上、三十人以上”是组织、领导传销活动罪的追诉门槛，不是多级返佣的安全线。
- 《禁止传销条例》明确把发展人员形成上下线，并以下线销售业绩给上线计酬列为高风险模式。
- 二级也不是当然合法；只要是上级按下级代理或间接客户销售业绩抽成，就仍然存在传销/违规分销风险。
- 本产品不申请直销牌照，也不做线下直销员体系；所以不按多级直销模式设计。
- 多级分销暂不启用；后续如业务确需二级或以上关系，必须先完成法务确认、单独方案评审和独立实现，不能在当前代理/邀请模块中顺手打开。

### 代理档位

代理先精简为一档，降低理解成本、财务结算复杂度和合规风险。

| 档位 | 门槛 | 直接订单现金佣金 | 推广积分 |
|---|---:|---:|---:|
| 默认代理 | 审核通过 | 20% | 不默认承诺；可由运营人工发放试用额度 |

说明：

- 直接客户指通过该代理链接/邀请码注册并绑定的客户。
- 客户首付、复购、后续充值或购买积分包，只要是该代理直接客户的有效订单，统一按 20% 现金佣金率计算。
- 复购佣金有效期：客户绑定代理后 180 天内有效。
- 实付金额不含赠送积分、优惠券抵扣、退款金额、平台补偿积分。
- 如运营发放推广积分，可即时到账，但不能提现、不参与佣金计算。
- 现金佣金 T+7 确认，月结提现。
- 不允许代理从下级代理或间接客户拿佣金。

后续如果代理规模起来，可以加“运营权益”或“服务商权限”，但仍不增加下级抽成层级。

### 代理客户归属

- 客户通过代理链接/邀请码注册后，绑定该代理。
- 首次绑定后 180 天内归属该代理。
- 若客户已经有代理，不允许被其他代理覆盖。
- 人工转移客户必须后台审批并记录。
- 代理自己注册的小号不计佣金。

### 结算规则

- 待结算：代理直接客户发生有效订单后实时产生。
- 可提现：T+7 后，无退款、无作弊、无争议。
- 提现门槛：100 元。
- 结算周期：每月 1 日-5 日结算上月可结算佣金；早期也可以人工每月导出。
- 平台可保留 5% 风控保证金，30 天后释放。
- T+7 内退款：佣金取消，积分奖励冻结或追回。
- 部分退款：按实际净支付金额重算佣金。
- 代理提现前必须完成实名/主体认证、代理协议确认、收款账户校验和必要税务信息留存。
- C 端积分不能提现；代理推广积分也不能提现，只有直推客户产生的现金佣金可申请提现。
- 当前本地 MVP 已有 `growth_service.py`、`commission_settlement_service.py` 和相关 API：支持创建代理档案、绑定直接客户、接受一级邀请；支付成功回调会写入代理直接客户 20% 待结算佣金，并对直接邀请首充发放 10% 积分奖励；退款回调会重算或取消未结算佣金，并追回多发的首充返利积分；后台可将 T+7 后佣金释放为可结算、创建结算批次并标记已支付。该能力仍不包含提现申请、实名、税务、真实财务打款、已打款后的财务追索和月度自动结算。

### 防作弊

- 同手机号、同支付账号、同设备、同 IP 高频互邀不计佣金。
- 代理给自己或关联账号充值/消费不计佣金。
- 异常退款率超过阈值，佣金进入冻结。
- 单客户异常高频生成、批量下载触发人工审核。
- 代理不能看到客户生成图原图，只能看订单、消耗和佣金。

### 代理后台 MVP

- 我的邀请码/推广链接。
- 今日新增客户、有效客户、客户实付金额、客户消耗积分。
- 待结算佣金、可提现佣金、推广积分。
- 客户列表：手机号脱敏、门店名、最近消耗、最近付款、状态。
- 佣金流水：来源客户、订单、实付金额、消耗积分、佣金金额、状态。
- 代理数据看板：访问量、注册数、付费数、佣金、转化率。

## C 端邀请返积分

C 端建议只返积分，不返现金。

原因：

- 普通客户的目标是降低出图成本，积分反馈更直接。
- 现金返利会引入提现、税务、刷单和更重风控。
- 积分不能提现，风险更低。

### 规则

邀请人奖励：

- 被邀请人注册成功：邀请人得 50 积分。
- 被邀请人首次有效充值：邀请人得首充实付金额对应积分的 10%。
- 邀请奖励积分有效期 90 天。

被邀请人奖励：

- 注册得 50 积分。

限制：

- 只算一级直接邀请，不做多级分销，不给间接邀请人返积分。
- 仅直接邀请的首充触发 10% 积分返利；复购、间接邀请、下级邀请都不触发 C 端返利。
- 注册必须使用手机号，并通过短信验证码。
- 注册奖励必须通过真人校验后才发放，例如图形验证码、滑块验证、或短信验证码 + 设备风控。
- 同设备、同手机号、同支付账号、同 IP 高频注册不发奖励。
- 同一手机号只能领取一次注册奖励。
- 同一设备 24 小时内最多 2 个新用户可触发注册奖励。
- 同一 IP 24 小时内最多 5 个新用户可触发注册奖励，超过进入人工审核或延迟发放。
- 高风险注册先进入 `frozen`，完成首次真实出图或人工审核后再发放。
- 被邀请人退款或订单作废，邀请奖励追回或冻结。
- 赠送积分不能提现，不参与代理佣金计算。
- C 端邀请只返积分，不返现金，不提供提现入口。
- 每个 C 端用户每月最多获得 1000 积分邀请奖励。
- C 端用户升级为代理后，后续邀请按代理规则计算，不再叠加 C 端奖励。

推荐展示文案：

```text
邀请好友用手机号完成注册，你和好友各得 50 积分；好友首充后，你得 10% 积分返利。
```

### 邀请状态

- `registered`: 已注册，待上传菜单。
- `registration_reward_pending`: 注册奖励待风控确认。
- `registration_reward_settled`: 邀请人和被邀请人各 50 积分已发放。
- `first_recharged`: 被邀请人已完成首次有效充值，10% 积分返利待风控确认。
- `first_recharge_rebate_settled`: 首充 10% 积分返利已发放。
- `frozen`: 风控冻结。
- `settled`: 当前可发奖励已发放。
